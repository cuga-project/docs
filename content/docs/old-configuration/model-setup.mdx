CUGA supports multiple LLM providers with flexible configuration options. You can configure models through TOML files or override specific settings using environment variables.

## Supported Platforms

- **OpenAI** - GPT models via OpenAI API (also supports LiteLLM via base URL override)
- **IBM WatsonX** - IBM's enterprise LLM platform
- **Azure OpenAI** - Microsoft's Azure OpenAI service
- **RITS** - Internal IBM research platform

## Configuration Priority

1. **Environment Variables** (highest priority)
   - `MODEL_NAME` - Override the model name
   - `AGENT_SETTING_CONFIG` - Override the configuration file
   - Provider-specific API keys and endpoints
2. **TOML Configuration** (medium priority)
3. **Default Values** (lowest priority)

### Option 1: OpenAI 🌐

**Setup Instructions:**

1. Create an account at [platform.openai.com](https://platform.openai.com)
2. Generate an API key from your [API keys page](https://platform.openai.com/api-keys)
3. Add to your `.env` file:
   ```bash
   # For OpenAI:
   # AGENT_SETTING_CONFIG="settings.openai.toml" #Could be replaced by the settings in ./configurations/models
   # OPENAI_API_KEY="XXXXXX"
   # OPENAI_API_VERSION="2024-08-01-preview"
   ```

**Default Values:**
- Model: `gpt-4o`
- API Version: OpenAI's default API Version
- Base URL: OpenAI's default endpoint

**Environment Variable Override:**
You can override the model name using:
```bash
MODEL_NAME="gpt-4-turbo"
```

### Option 2: IBM WatsonX 🔵

**Setup Instructions:**

1. Access [IBM WatsonX](https://www.ibm.com/watsonx)
2. Create a project and get your credentials:
   - Project ID
   - API Key
   - Region/URL
3. Add to your `.env` file:
   ```bash
   # For WatsonX
   # AGENT_SETTING_CONFIG="settings.watsonx.toml" #Could be replaced by the settings in ./configurations/models
   # WATSONX_PROJECT_ID="XXXXXX"
   # WATSONX_URL="https://us-south.ml.cloud.ibm.com"
   # WATSONX_APIKEY="XXXXXX"
   ```

**Default Values:**
- Model: `meta-llama/llama-4-maverick-17b-128e-instruct-fp8`

**Environment Variable Override:**
You can override the model name using:
```bash
MODEL_NAME="meta-llama/llama-3.1-8b-instruct"
```

### Option 3: Azure OpenAI

**Setup Instructions:**

1. Add to your `.env` file:
   ```bash
   # For Azure Openai
   # AGENT_SETTING_CONFIG="settings.azure.toml" #Could be replaced by the settings in ./configurations/models
   # AZURE_OPENAI_API_KEY="XXXXXX"
   # AZURE_OPENAI_ENDPOINT="XXXXXX"
   ```

**Environment Variable Override:**
You can override the model name using:
```bash
MODEL_NAME="gpt-4o"
```

### LiteLLM Support

CUGA supports LiteLLM through the OpenAI configuration by overriding the base URL:

1. Add to your `.env` file:
   ```bash
   # For LiteLLM:
   # AGENT_SETTING_CONFIG="settings.openai.toml" #Could be replaced by the settings in ./configurations/models
   # OPENAI_API_KEY="XXXXXX"
   # OPENAI_BASE_URL="XXXXXX"
   ```

**Environment Variable Override:**
You can override the model name using:
```bash
MODEL_NAME="gpt-4-turbo"
```

## Configuration Files

CUGA uses TOML configuration files located in `src/cuga/configurations/models/`:

- `settings.openai.toml` - OpenAI configuration (also supports LiteLLM via base URL override)
- `settings.watsonx.toml` - WatsonX configuration  
- `settings.azure.toml` - Azure OpenAI configuration

Each file contains agent-specific model settings that can be overridden by environment variables.

### Settings Structure 

```
configurations/
└── models/
    ├── settings.litellm.toml      # Default ETE configuration
    ├── settings.openai.toml       # OpenAI configuration
    ├── settings.watsonx.toml      # WatsonX configuration
    └── settings.azure.toml        # Azure configuration
```

### Settings File Content Structure

Each settings file contains configuration for different CUGA agents. You can configure each agent independently with different models and parameters:

#### OpenAI Configuration Example

```toml
[agent.task_decomposition.model]
platform = "openai"
temperature = 0.1
max_tokens = 1000

[agent.planner.model]
platform = "openai"
temperature = 0.1
max_tokens = 5000

[agent.chat.model]
platform = "openai"
temperature = 0.1
max_tokens = 5000

[agent.shortlister.model]
platform = "openai"
temperature = 0.1
max_tokens = 7000
```

#### WatsonX Configuration Example

```toml
[agent.shortlister.model]
platform = "watsonx"
model_name = "meta-llama/llama-4-maverick-17b-128e-instruct-fp8"
temperature = 0.1
max_tokens = 7000

[agent.planner.model]
platform = "watsonx"
model_name = "meta-llama/llama-4-maverick-17b-128e-instruct-fp8"
temperature = 0.1
max_tokens = 5000

[agent.chat.model]
platform = "watsonx"
model_name = "meta-llama/llama-4-maverick-17b-128e-instruct-fp8"
temperature = 0.1
max_tokens = 5000
```

#### Agent Types

CUGA uses different specialized agents for different tasks:

- **`task_decomposition`**: Breaks down complex tasks into smaller, manageable steps
- **`planner`**: Creates execution plans and strategies
- **`chat`**: Handles conversational interactions and responses
- **`shortlister`**: Prioritizes and filters tasks or options

Each agent can be configured with:
- **`platform`**: The LLM provider (openai, watsonx, azure)
- **`model_name`**: Specific model to use (optional, uses platform default if not specified)
- **`temperature`**: Controls randomness (0.0 = deterministic, 1.0 = very random)
- **`max_tokens`**: Maximum number of tokens in the response

### Example Configuration Files

#### settings.litellm.toml (Default)
```toml
[llm]
provider = "litellm"
model = "gpt-4"
temperature = 0.1
max_tokens = 4000

[litellm]
base_url = "https://ete-litellm.bx.cloud9.ibm.com"
```

#### settings.openai.toml
```toml
[llm]
provider = "openai"
model = "gpt-4"
temperature = 0.1
max_tokens = 4000
```

## 🔄 Switching Between Providers

### Method 1: Environment Variable

```bash
# Switch to OpenAI
export AGENT_SETTING_CONFIG="settings.openai.toml"

# Switch to WatsonX
export AGENT_SETTING_CONFIG="settings.watsonx.toml"

# Switch to Azure
export AGENT_SETTING_CONFIG="settings.azure.toml"
```

### Method 2: Edit .env File

```bash
# Edit .env file
AGENT_SETTING_CONFIG="settings.openai.toml"
```
