# ✨ **CUGA Runtime Tools Integration** ✨

CUGA supports loading and running **LangChain tools** dynamically during runtime.

## ✅ **What You’ll Do**

*   🛠 **Write** runtime tools in Python.
*   🔗 **Integrate** them into CUGA for dynamic task execution.

***

## **1️⃣ Writing a Tool**

Create your tool as a standard Python function and wrap it as a **LangChain Structured Tool**. Then, load it into CUGA in your runtime code.

**Example:**

```python
from langchain_core.tools import StructuredTool

def foo():
    print("hello")

tools = [foo]
```

***

## **2️⃣ Adding It to CUGA**
Dynamically add the created tools in your code, using the tracker to set the tools.

Here’s an example of how to integrate your tools into CUGA:

```python
cuga_agent = CugaAgent(browser_enabled=False)
await cuga_agent.initialize_appworld_env()

tools = [foo]
for tool in tools:
    tool.metadata = {'server_name': "test"}

tracker.set_tools(tools)

task_result: AgentResult = await cuga_agent.run_task_generic(
    eval_mode=False,
    goal=task
)
```

📂 Full example available in:

    docs/examples/cuga_with_runtime_tools

***

## **3️⃣ What Happens When You Run CUGA**

*   📦 **Loads all APIs and MCP servers** from `mcp_servers.yaml`.
*   🔄 **Loads your LangChain tools** during runtime.
*   ▶️ **Executes the task** defined in your code.
*   🖥 **Displays logs** in the terminal for real-time tracking.
