## 🚀 Testing Overview

### Testing Pyramid

CUGA testing follows a comprehensive testing strategy:

```
    🔺 E2E Tests (Few, Critical)
   🔺🔺 Integration Tests (Some, Important)
  🔺🔺🔺 Unit Tests (Many, Fast)
 🔺🔺🔺🔺 Component Tests (Many, Isolated)
```

### Test Categories

- **Unit Tests**: Individual component testing
- **Integration Tests**: Component interaction testing
- **System Tests**: End-to-end workflow testing
- **Performance Tests**: Load and stress testing
- **Benchmark Tests**: Evaluation framework testing

## 🧪 Running Tests

### Basic Test Execution

```bash
# Run all tests
uv run pytest

# Run with verbose output
uv run pytest -v

# Run specific test file
uv run pytest tests/test_agent.py

# Run specific test function
uv run pytest tests/test_agent.py::test_agent_initialization
```

### Test Suites

```bash
# Run unit tests only
uv run pytest tests/unit/

# Run integration tests only
uv run pytest tests/integration/

# Run system tests only
uv run pytest tests/system/

# Run all tests with coverage
uv run pytest --cov=cuga --cov-report=html
```

### Test Configuration

```bash
# Run tests with specific configuration
uv run pytest --config=test_config.toml

# Run tests in parallel
uv run pytest -n auto

# Run tests with specific markers
uv run pytest -m "slow"
uv run pytest -m "not slow"
```

## 🔧 Unit Testing

### Agent Testing

Test individual agent components:

```python
# tests/test_agent.py
import pytest
from cuga.agent import Agent
from cuga.config import Config

class TestAgent:
    def test_agent_initialization(self):
        """Test agent initialization with valid config"""
        config = Config()
        agent = Agent(config)
        assert agent is not None
        assert agent.config == config
    
    def test_agent_temperature_setting(self):
        """Test agent temperature configuration"""
        config = Config()
        config.set('agent.temperature', 0.5)
        agent = Agent(config)
        assert agent.temperature == 0.5
    
    def test_agent_invalid_config(self):
        """Test agent initialization with invalid config"""
        with pytest.raises(ValueError):
            agent = Agent(None)
```

### Configuration Testing

Test configuration management:

```python
# tests/test_config.py
import pytest
from cuga.config import Config

class TestConfig:
    def test_config_loading(self):
        """Test configuration file loading"""
        config = Config.from_file("tests/fixtures/test_config.toml")
        assert config.get('agent.temperature') == 0.1
    
    def test_config_validation(self):
        """Test configuration validation"""
        config = Config()
        config.set('invalid.setting', 'value')
        
        with pytest.raises(ValueError):
            config.validate()
    
    def test_config_export(self):
        """Test configuration export"""
        config = Config()
        config.set('agent.temperature', 0.2)
        
        exported = config.export('toml')
        assert 'temperature = 0.2' in exported
```

### Tool Testing

Test individual tools and functions:

```python
# tests/test_tools.py
import pytest
from cuga.tools import ToolRegistry, Tool

class TestTools:
    def test_tool_registration(self):
        """Test tool registration"""

        tool = Tool("test_tool", "Test tool description")
        
        registry.register(tool)
        assert registry.get("test_tool") == tool

        registry = ToolRegistry()

        tool = Tool("test_tool", "Test tool", test_function)
        registry.register(tool)
        
        result = registry.execute("test_tool")
        assert result == "test_result"
```

## 🔗 Integration Testing

### API Integration Testing

Test API interactions:

```python
# tests/test_api_integration.py
import pytest
import requests
from cuga.api import APIClient

class TestAPIIntegration:
    @pytest.fixture
    def api_client(self):
        """Create API client for testing"""
        return APIClient("http://localhost:8005")
    
    def test_api_health_check(self, api_client):
        """Test API health endpoint"""
        response = api_client.health_check()
        assert response.status_code == 200
        assert response.json()["status"] == "healthy"
    
    def test_api_chat_endpoint(self, api_client):
        """Test chat endpoint"""
        response = api_client.chat("Hello, CUGA!")
        assert response.status_code == 200
        assert "response" in response.json()
    
    def test_api_error_handling(self, api_client):
        """Test API error handling"""
        with pytest.raises(requests.exceptions.RequestException):
            api_client.chat("")  # Empty message should fail
```

## 🧪 System Testing

```python
import pytest
from cuga.system import SystemTest

class TestEndToEnd:
    def test_demo_workflow(self):

        system = SystemTest()
        
        # Start demo environment
        system.start_demo()
        
        # Execute test query
        result = system.execute_query("get top account by revenue")
        
        # Verify result

        system.cleanup()

        """Test system error recovery"""
        system = SystemTest()
        
        # Introduce error
        system.introduce_error()
        
        # Verify recovery
        assert system.is_healthy()
        
        # Cleanup
        system.cleanup()
```

```python
import pytest
from cuga.workflows import WorkflowEngine

class TestWorkflows:
    def test_workflow_execution(self):

        # Load workflow
        loaded_workflow = engine.load(workflow_id)
        assert loaded_workflow.name == "persistent_workflow"
```

## 📊 Performance Testing

### Load Testing

Test system performance under load:

```python
# tests/test_performance.py
import pytest
import time
from cuga.performance import LoadTester

class TestPerformance:
    def test_response_time(self):
        """Test response time under normal load"""
        tester = LoadTester()
        
        # Measure response time
        start_time = time.time()
        result = tester.execute_query("get top account by revenue")
        end_time = time.time()
        
        response_time = end_time - start_time
        assert response_time < 5.0  # Should complete within 5 seconds
        assert result.success
    
    def test_concurrent_requests(self):
        """Test system under concurrent load"""
        tester = LoadTester()
        
        # Execute multiple concurrent requests
        results = tester.concurrent_requests(
            query="get top account by revenue",
            count=10
        )
        
        # Verify all requests succeeded
        assert all(r.success for r in results)
        assert len(results) == 10
    
    def test_memory_usage(self):
        """Test memory usage under load"""
        tester = LoadTester()
        
        # Monitor memory usage
        memory_before = tester.get_memory_usage()
        
        # Execute multiple requests
        tester.execute_multiple_requests(100)
        
        memory_after = tester.get_memory_usage()
        memory_increase = memory_after - memory_before
        
        # Memory increase should be reasonable
        assert memory_increase < 100 * 1024 * 1024  # Less than 100MB
```

### Stress Testing

Test system limits:

```python
# tests/test_stress.py
import pytest
from cuga.performance import StressTester

class TestStress:
    def test_max_concurrent_connections(self):

        tester = StressTester()

        # System should handle gracefully
        assert len(connections) <= 100  # Max connections
        assert all(c.is_healthy() for c in connections)
    
    def test_large_payload_handling(self):
        """Test handling of large payloads"""
        tester = StressTester()
        
        # Create large payload
        large_payload = "x" * (10 * 1024 * 1024)  # 10MB
        
        # System should handle gracefully
        result = tester.send_large_payload(large_payload)
        assert result.success
```

## 🚨 Test Troubleshooting

### Common Test Issues

| Issue | Solution |
|-------|----------|
| 🔴 Tests failing randomly | Use `--randomly-seed` for reproducible results |
| 🔴 Slow test execution | Use `-n auto` for parallel execution |
| 🔴 Memory leaks in tests | Use `--maxfail=1` to stop on first failure |
| 🔴 External dependencies | Mock external services in tests |
| 🔴 Configuration conflicts | Use separate test configuration files |

### Debug Commands

```bash
# Run tests with debug output
uv run pytest -v -s --tb=long

# Run specific failing test
uv run pytest tests/test_agent.py::test_agent_initialization -v -s

# Run tests with specific markers
uv run pytest -m "slow" -v

# Run tests with coverage and stop on failure
uv run pytest --cov=cuga --maxfail=1
```

## 📚 Best Practices

### Test Design

1. **Test Naming**: Use descriptive test names that explain the scenario
2. **Test Isolation**: Each test should be independent and not affect others
3. **Test Data**: Use fixtures and factories for test data
4. **Assertions**: Make assertions specific and meaningful
5. **Error Testing**: Test both success and failure scenarios

### Test Organization

1. **Test Structure**: Organize tests by component and functionality
2. **Test Categories**: Separate unit, integration, and system tests
3. **Test Data**: Use separate test data and configuration
4. **Test Utilities**: Create helper functions for common test operations
5. **Test Documentation**: Document complex test scenarios

### Test Maintenance

1. **Regular Updates**: Update tests when functionality changes
2. **Test Coverage**: Maintain high test coverage for critical components
3. **Performance Monitoring**: Monitor test execution time and optimize
4. **Test Reliability**: Ensure tests are stable and don't flake
5. **Test Review**: Regularly review and refactor tests
