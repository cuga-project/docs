# Installation Guide

This comprehensive guide will walk you through installing CUGA on your system, including all dependencies and configuration options. CUGA is an enterprise agent you can trust for building reliable agents for real-world use.

## System Requirements

<Callout type="info">
CUGA requires Python 3.12 or higher and works on Windows, macOS, and Linux systems.
</Callout>

### Minimum Requirements
- **Python**: 3.12 or higher
- **RAM**: 8GB minimum, 16GB recommended
- **Storage**: 2GB free space
- **Network**: Internet connection for API access

### Recommended Setup
- **Python**: 3.12+
- **RAM**: 16GB or more
- **Storage**: 5GB free space
- **OS**: Ubuntu 20.04+, macOS 12+, Windows 10+

## Prerequisites

Before installing CUGA, ensure you have these tools:

<Steps>
  <Step title="Python 3.12+">
    Install Python 3.12 or higher from python.org
  </Step>
  <Step title="UV Package Manager">
    Install UV for modern Python dependency management
  </Step>
  <Step title="Git">
    Install Git for version control and cloning
  </Step>
  <Step title="API Keys">
    Obtain necessary API keys for LLM services
  </Step>
</Steps>

### Installing UV

UV is the recommended package manager for CUGA:

```bash
# macOS/Linux
curl -LsSf https://astral.sh/uv/install.sh | sh

# Windows (PowerShell)
powershell -c "irm https://astral.sh/uv/install.ps1 | iex"

# Or via pip
pip install uv
```

<Callout type="warning">
UV provides faster dependency resolution and better virtual environment management than pip.
</Callout>

  <TabsList>
    <TabsTrigger value="standard">Standard Installation</TabsTrigger>

    <TabsTrigger value="development">Development Setup</TabsTrigger>

  <TabsContent value="standard">
    <h3>Standard Installation</h3>
    <p>Recommended for most users. Install CUGA directly on your system.</p>
    
    <Steps>
      <Step title="Clone Repository">
        ```bash
        git clone https://github.com/cuga-project/cuga-agent.git
        cd cuga-agent
        ```
      </Step>
      <Step title="Create Environment">
        ```bash
        uv venv --python=3.12
        source .venv/bin/activate  # Windows: .venv\Scripts\activate
        ```
      </Step>
      <Step title="Install Dependencies">
        ```bash
        uv sync
        ```
      </Step>
      <Step title="Configure Environment">
        ```bash
        cp .env.example .env
        # Edit .env with your API keys
        ```
      </Step>
    </Steps>
  </TabsContent>
  
  <TabsContent value="docker">
    <h3>Docker Installation</h3>
    <p>Use Docker for isolated, reproducible environments.</p>
    
    <Callout type="info">
    Docker installation is recommended for evaluation and testing purposes.
    </Callout>
    
    ```bash
    # Pull the CUGA Docker image
    docker pull ibmresearch/cuga:latest
    
    # Run CUGA container
    docker run -it --rm \
      -p 8000:8000 \
      -e OPENAI_API_KEY=your_key_here \
      ibmresearch/cuga:latest
    ```
  </TabsContent>
  
  <TabsContent value="development">
    <h3>Development Setup</h3>
    <p>For contributors and developers working on CUGA.</p>
    
    ```bash
    # Clone with submodules
    git clone --recursive https://github.com/cuga-project/cuga-agent.git
    cd cuga-agent
    
    # Install in development mode
    uv sync --dev
    
    # Install pre-commit hooks
    pre-commit install
    ```
  </TabsContent>
</Tabs>

## Configuration

### Environment Variables

Create a `.env` file in your CUGA directory:

<CodeBlock lang="bash">
# Required: Your OpenAI API key
OPENAI_API_KEY=sk-your-key-here

# Optional: Model configuration
AGENT_SETTING_CONFIG=settings.openai.toml

# Optional: Port configuration
CUGA_PORT=8000
CUGA_DASHBOARD_PORT=8005
</CodeBlock>

### Model Configuration

CUGA supports multiple LLM providers:

<Tabs defaultValue="openai">
  <TabsList>
    <TabsTrigger value="openai">OpenAI</TabsTrigger>
    <TabsTrigger value="ete">ETE LiteLLM</TabsTrigger>
    <TabsTrigger value="watsonx">IBM WatsonX</TabsTrigger>
    <TabsTrigger value="azure">Azure OpenAI</TabsTrigger>
  </TabsList>
  
  <TabsContent value="openai">
    <h4>OpenAI Configuration</h4>
    <p>Use OpenAI's GPT models directly:</p>
    
    ```bash
    OPENAI_API_KEY=sk-your-openai-key
    AGENT_SETTING_CONFIG=settings.openai.toml
    ```
  </TabsContent>
  
  <TabsContent value="ete">
    <h4>ETE LiteLLM Configuration</h4>
    <p>IBM internal LLM service (recommended for team members):</p>
    
    ```bash
    OPENAI_API_KEY=ete-your-ete-key
    AGENT_SETTING_CONFIG=settings.litellm.toml
    ```
    
    <Callout type="info">
    Get your ETE key from the [ETE LiteLLM portal](https://ete-litellm.bx.cloud9.ibm.com/ui/).
    </Callout>
  </TabsContent>
  
  <TabsContent value="watsonx">
    <h4>IBM WatsonX Configuration</h4>
    <p>Use IBM's WatsonX models:</p>
    
    ```bash
    WATSONX_API_KEY=your-watsonx-key
    WATSONX_PROJECT_ID=your-project-id
    WATSONX_URL=https://us-south.ml.cloud.ibm.com
    AGENT_SETTING_CONFIG=settings.watsonx.toml
    ```
  </TabsContent>
  
  <TabsContent value="azure">
    <h4>Azure OpenAI Configuration</h4>
    <p>Use Azure-hosted OpenAI models:</p>
    
    ```bash

    AZURE_OPENAI_ENDPOINT=your-azure-endpoint
    OPENAI_API_VERSION=2024-08-01-preview
    AGENT_SETTING_CONFIG=settings.azure.toml
    ```
  </TabsContent>
</Tabs>

## Verification

After installation, verify everything is working:

```bash
# Test the code sandbox
uv run test_sandbox

# Expected output: ('test succeeded\n', {})
```

<Callout type="success">
If you see the success message, CUGA is properly installed and configured!
</Callout>

## First Run

Start CUGA for the first time:

```bash
# Start in demo mode
cuga start demo

# Or start the main service
cuga start
```

## Troubleshooting

### Common Issues

<Callout type="warning">
Most installation issues are related to Python version, dependencies, or API key configuration.
</Callout>

| Issue | Solution |
|-------|----------|
| **Python version error** | Ensure Python 3.12+ is installed and in PATH |
| **Module not found** | Run `uv sync` to install dependencies |
| **API key errors** | Verify your `.env` file has correct API keys |
| **Port conflicts** | Check if ports 8000/8005 are available |
