Master CUGA's command-line interface for service management, monitoring, and control.

## 🚀 Command Overview

CUGA provides a comprehensive CLI for managing all aspects of the system:

```bash
# Basic command structure
cuga <command> <subcommand> [options]
```

## 📋 Available Commands

### Service Management

| Command | Description | Example |
|---------|-------------|---------|
| `start` | Start CUGA services | `cuga start demo` |
| `stop` | Stop CUGA services | `cuga stop all` |
| `status` | Check service status | `cuga status` |
| `restart` | Restart services | `cuga restart demo` |

### Configuration

| Command | Description | Example |
|---------|-------------|---------|
| `config` | Manage configuration | `cuga config show` |
| `env` | Environment management | `cuga env list` |
| `mode` | Switch execution modes | `cuga mode fast` |

### Monitoring

| Command | Description | Example |
|---------|-------------|---------|
| `exp` | Experiment dashboard | `cuga exp` |
| `logs` | View system logs | `cuga logs --tail` |
| `monitor` | Real-time monitoring | `cuga monitor` |

## 🚀 Service Management

### Starting Services

#### Start Demo Mode
```bash
# Start complete demo environment
cuga start demo

# Start specific demo components
cuga start api-registry
cuga start cuga-demo
```

#### Start Evaluation Mode
```bash
# Start AppWorld evaluation
cuga start appworld

# Start WebArena evaluation
cuga start webarena

# Start WxO tools
cuga start wxo
```

#### Start Custom Services
```bash
# Start with specific configuration
cuga start demo --config settings.fast.toml

# Start on specific port
cuga start demo --port 8006

# Start with custom environment
cuga start demo --env production
```

### Stopping Services

#### Stop All Services
```bash
# Stop all running services
cuga stop all

# Force stop (kill processes)
cuga stop all --force
```

#### Stop Specific Services
```bash
# Stop demo services
cuga stop demo

# Stop evaluation services
cuga stop appworld
cuga stop webarena

# Stop by port
cuga stop --port 8005
```

### Service Status

#### Check All Services
```bash
# View status of all services
cuga status

# Detailed status with logs
cuga status --verbose

# Status in JSON format
cuga status --format json
```

#### Check Specific Services
```bash
# Check demo status
cuga status demo

# Check by service type
cuga status --type api
cuga status --type evaluation
```

## ⚙️ Configuration Management

### View Configuration

#### Show Current Settings
```bash
# Show all configuration
cuga config show

# Show specific sections
cuga config show --section agent
cuga config show --section features
cuga config show --section execution
```

#### Export Configuration
```bash
# Export to different formats
cuga config export --format toml
cuga config export --format json
cuga config export --format yaml

# Export specific sections
cuga config export --section agent --format json
```

### Modify Configuration

#### Update Settings
```bash
# Update specific values
cuga config set agent.temperature 0.2
cuga config set features.mode hybrid
cuga config set execution.max_retries 5

# Update from file
cuga config import --file custom_settings.toml

# Reset to defaults
cuga config reset
```

#### Validate Configuration
```bash
# Validate current configuration
cuga config validate

# Validate specific file
cuga config validate --file settings.toml

# Check for conflicts
cuga config validate --check-conflicts
```

## 🔄 Mode Management

### Switch Execution Modes

#### Fast Mode
```bash
# Switch to fast mode
cuga mode fast

# Verify mode change
cuga config show --section features
```

#### Accurate Mode
```bash
# Switch to accurate mode
cuga mode accurate

# Start demo with accurate mode
cuga start demo --mode accurate
```

#### Custom Mode
```bash
# Switch to custom mode
cuga mode custom

# Use custom configuration file
cuga mode custom --config my_custom.toml
```

### Mode Information
```bash
# List available modes
cuga mode list

# Show current mode
cuga mode current

# Show mode details
cuga mode info fast
```

## 📊 Monitoring Commands

### Experiment Dashboard

#### Launch Dashboard
```bash
# Open experiment dashboard
cuga exp

# Dashboard with specific experiment
cuga exp --experiment my_experiment

# Dashboard in browser
cuga exp --browser
```

#### Experiment Management
```bash
# List experiments
cuga exp list

# Show experiment details
cuga exp show my_experiment

# Delete experiment
cuga exp delete my_experiment
```

### Log Management

#### View Logs
```bash
# View recent logs
cuga logs

# Follow logs in real-time
cuga logs --tail

# View specific log files
cuga logs --file demo.log
cuga logs --file error.log
```

#### Log Filtering
```bash
# Filter by level
cuga logs --level ERROR
cuga logs --level INFO,WARNING

# Filter by time
cuga logs --since "2024-01-01"
cuga logs --until "2024-01-31"

# Filter by service
cuga logs --service demo
cuga logs --service api-registry
```

### System Monitoring

#### Real-time Monitoring
```bash
# Start monitoring dashboard
cuga monitor

# Monitor specific metrics
cuga monitor --metrics cpu,memory,network

# Monitor specific services
cuga monitor --services demo,api-registry
```

#### Performance Metrics
```bash
# Show performance stats
cuga stats

# Show resource usage
cuga stats --resources

# Show API metrics
cuga stats --api
```

## 🔧 Advanced Commands

### Environment Management

#### Environment Variables
```bash
# List environment variables
cuga env list

# Set environment variable
cuga env set OPENAI_API_KEY new_key_here

# Unset environment variable
cuga env unset DEBUG_MODE

# Export environment
cuga env export --file .env.backup
```

#### Environment Profiles
```bash
# Create profile
cuga env profile create development

# Switch profile
cuga env profile use production

# List profiles
cuga env profile list
```

### Service Discovery

#### Available Services
```bash
# List available services
cuga services list

# Show service details
cuga services info demo

# Check service health
cuga services health
```

#### Service Dependencies
```bash
# Show service dependencies
cuga services deps

# Check dependency health
cuga services deps --health

# Resolve dependencies
cuga services deps --resolve
```

## 🧪 Testing Commands

### System Testing

#### Health Checks
```bash
# Run health check
cuga health

# Health check for specific service
cuga health --service demo

# Detailed health report
cuga health --verbose
```

#### Integration Tests
```bash
# Run integration tests
cuga test integration

# Test specific components
cuga test --component api
cuga test --component demo

# Test with coverage
cuga test --coverage
```

### Performance Testing

#### Benchmark Tests
```bash
# Run benchmarks
cuga benchmark

# Benchmark specific mode
cuga benchmark --mode fast
cuga benchmark --mode accurate

# Compare modes
cuga benchmark --compare fast,accurate,custom
```

#### Load Testing
```bash
# Load test demo
cuga load-test --service demo --users 100

# Stress test API
cuga load-test --service api --duration 300
```


## 🐛 Troubleshooting

### Common Command Issues

| Issue                    | Solution                                    |
| ------------------------ | ------------------------------------------- |
| 🔴 Command not found     | Check CUGA installation and PATH            |
| 🔴 Permission denied     | Use sudo or check file permissions          |
| 🔴 Service not responding | Check service status and restart if needed  |
| 🔴 Configuration error    | Validate configuration with `cuga config validate` |
| 🔴 Port conflicts        | Check port usage and stop conflicting services |

```bash
cuga env list
```
