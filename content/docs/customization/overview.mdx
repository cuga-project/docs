You can customize CUGA's behavior and features through the use of configuration files and environment variables.

## Configuration Structure

By default, CUGA uses configuration files which are stored in the `src/cuga/configuration/` folder, however you can
specify a custom configuration directory via this environment variable:

<Tabs defaultValue="mac">
  <TabsList>
    <TabsTrigger value="mac">Mac</TabsTrigger>
    <TabsTrigger value="windows">Windows</TabsTrigger>
  </TabsList>

  <TabsContent value="mac">
    ```bash
    # Set custom configuration directory
    export CUGA_CONFIGURATIONS_DIR="/path/to/your/configurations"
    ```

  </TabsContent>

  <TabsContent value="windows">
    ```powershell
    # Set custom configuration directory
    setx CUGA_CONFIGURATIONS_DIR "C:\path\to\your\configurations"
    ```

  </TabsContent>
</Tabs>

This allows you to organize multiple configurations in separate folders.

Each configuration folder, contains TOML and MD files that allow to toggle features and customize behavior.

```
configurations/
├── models/
│   ├── settings.litellm.toml      # Default ETE configuration
│   ├── settings.openai.toml       # OpenAI configuration
│   ├── settings.watsonx.toml      # WatsonX configuration
│   └── settings.azure.toml        # Azure configuration
├── modes/
│   ├── fast.toml              # Fast mode settings
│   ├── balanced.toml          # Balanced mode settings
│   └── accurate.toml          # Accurate mode settings
└── instructions/
    └── default/
        ├── api_planner.md     # Custom API planner instructions
        ├── task_decomposition.md   # Custom task decomposition instructions
        └── code_agent.md  # Custom code agent instructions
```

## 🚀 Reasoning Modes

### Mode Selection

CUGA offers three reasoning modes optimized for different use cases:

| Mode         | Speed      | Accuracy   | Use Case                   |
| ------------ | ---------- | ---------- | -------------------------- |
| **Fast**     | ⭐⭐⭐⭐⭐ | ⭐⭐⭐     | Development, testing       |
| **Balanced** | ⭐⭐⭐⭐   | ⭐⭐⭐⭐   | General purpose, workflows |
| **Accurate** | ⭐⭐⭐     | ⭐⭐⭐⭐⭐ | Production, critical tasks |

### Fast Mode Configuration

```toml
# configurations/modes/fast.toml
[features]
chat = true
task_decomposition = false
local_sandbox = true
thoughts = false
code_output_summary = false
code_generation = "fast"
code_output_reflection = false
save_reuse = false
forced_apps = []
final_answer = true
```

**Fast Mode Features:**

-   **Speed Optimized**: Minimal task decomposition for quick execution
-   **Direct Execution**: Bypasses complex reasoning chains
-   **Local Sandbox**: Uses local execution environment
-   **No Reflection**: Skips output validation for speed
-   **Quick Code Generation**: Fast code generation without extensive validation

### Balanced Mode Configuration

```toml
# configurations/modes/balanced.toml
[features]
chat = false
task_decomposition = true
local_sandbox = true
thoughts = false
code_output_summary = false
code_generation = "fast"
code_output_reflection = true
save_reuse = false
forced_apps = []
final_answer = true
```

**Balanced Mode Features:**

-   **Task Decomposition**: Breaks complex tasks into manageable steps
-   **Reflection Enabled**: Validates code output for quality
-   **Local Sandbox**: Uses local execution environment
-   **No Chat Interface**: Focuses on API-based execution
-   **Fast Code Generation**: Quick code generation with validation

### Accurate Mode Configuration

```toml
# configurations/modes/accurate.toml
[features]
chat = false
task_decomposition = true
local_sandbox = true
thoughts = true
code_output_summary = true
code_generation = "accurate"
code_output_reflection = true
save_reuse = true
forced_apps = []
final_answer = true
```

**Accurate Mode Features:**

-   **Full Task Decomposition**: Comprehensive task breakdown
-   **Thought Process**: Enables detailed reasoning chains
-   **Code Output Summary**: Provides detailed execution summaries
-   **Accurate Code Generation**: High-quality code with extensive validation
-   **Save & Reuse**: Enables workflow persistence
-   **Multiple Reflections**: Multiple validation passes for accuracy

## 📝 Custom Instructions

### File System Configuration

CUGA supports custom instructions through markdown files in the configuration directory:

```
configurations/instructions/default/
├── api_code_planner.md     # API code planning instructions
├── plan_controller.md      # Plan controller instructions
├── task_decomposition.md   # Task decomposition instructions
├── api_planner.md          # API planning instructions
├── answer.md               # Final answer instructions
├── shortlister.md          # Shortlister instructions
└── code_agent.md           # Code generation instructions
```

### Custom Instruction Structure

Each instruction file should contain markdown content that defines the behavior for specific components:

```markdown
# API Planner Instructions

## Role

You are an expert API planner responsible for breaking down complex tasks into API calls.

## Guidelines

-   Always validate API endpoints before use
-   Consider rate limits and authentication
-   Provide clear error handling strategies
-   Document expected responses

## Examples

-   For data retrieval: Use GET requests with proper query parameters
-   For data modification: Use POST/PUT requests with validation
```

### Instruction File Naming

Custom instruction files follow the pattern:

-   `./configurations/instructions/default/api_code_planner.md`
-   `./configurations/instructions/default/plan_controller.md`
-   `./configurations/instructions/default/task_decomposition.md`
-   `./configurations/instructions/default/api_planner.md`
-   `./configurations/instructions/default/answer.md`
-   `./configurations/instructions/default/shortlister.md`
-   `./configurations/instructions/default/code_agent.md`

## 🎮 Mode Switching

### Method 1: Environment Variable

```bash
# Switch to fast mode
export DYNACONF_FEATURES__CUGA_MODE="fast"

# Switch to balanced mode
export DYNACONF_FEATURES__CUGA_MODE="balanced"

# Switch to accurate mode
export DYNACONF_FEATURES__CUGA_MODE="accurate"
```

### Method 2: Settings File

Edit `settings.toml`:

```toml
[features]
cuga_mode = "balanced"  # or "fast" or "accurate"
```
