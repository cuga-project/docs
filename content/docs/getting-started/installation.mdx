# Installation Guide

Get CUGA up and running on your system in just a few minutes. This comprehensive guide will walk you through installing CUGA, from quick setup to detailed configuration options. CUGA is an enterprise agent you can trust for building reliable agents for real-world use.

## Prerequisites

### System Requirements

<Callout type="info">
CUGA requires Python 3.12 or higher and works on Windows, macOS, and Linux systems.
</Callout>

- **Python**: 3.12 or higher
- **RAM**: 8GB minimum, 16GB recommended
- **Storage**: 2GB free space (5GB for development)
- **Network**: Internet connection for API access
- **OS**: Ubuntu 20.04+, macOS 12+, or Windows 10+

### Required Tools

Before you begin, ensure you have the following:

- **UV Package Manager** - Modern Python dependency management (see [installation guide](#installing-uv) below)
- **Git** - For cloning the repository
- **API Keys** - OpenAI API key or compatible LLM provider

<Callout type="warning">
Additional requirements vary by installation method. See the specific installation section below for details.
</Callout>

## 🚀 Quick Start

Get started with the essential setup steps to have CUGA running quickly using the standard installation method.

<Steps>
  <Step title="Clone Repository">
    Clone the CUGA repository to your local machine:

    ```bash
    git clone https://github.com/cuga-project/cuga-agent.git
    cd cuga-agent
    ```
  </Step>

  <Step title="Setup Environment">
    Create a virtual environment and install dependencies:

    ```bash
    uv venv --python=3.12
    source .venv/bin/activate  # On Windows: .venv\Scripts\activate
    uv sync
    ```

    <Callout type="warning">
    Make sure to activate the virtual environment before running any CUGA commands.
    </Callout>
  </Step>

  <Step title="Configure Environment">
    Copy the example environment file and add your API key:

    ```bash
    cp .env.example .env
    ```

    Edit the `.env` file and add your OpenAI API key:

    ```bash
    OPENAI_API_KEY=your_api_key_here
    ```

    <Callout type="info">
    For IBM team members, you can use the ETE LiteLLM API key from the [ETE LiteLLM portal](https://ete-litellm.bx.cloud9.ibm.com/ui/).
    </Callout>
  </Step>

  <Step title="Test Installation">
    Verify that CUGA is working correctly:

    ```bash
    uv run test-sandbox
    ```

    If everything is set up correctly, you should see a success message: `('test succeeded\n', {})`
  </Step>
</Steps>

### First Run

Start CUGA in demo mode to explore its capabilities:

```bash
cuga start demo
```

Try this example query to test the system:

```text
get my top account by revenue from digital sales
```

        ```
      </Step>

        uv venv --python=3.12

        ```
      </Step>

  <TabsContent value="docker">
    <h3>Docker Installation</h3>
    <p>Use Docker for isolated, reproducible environments.</p>

    **Requirements:**
    - Docker Desktop or Docker Engine
    - 4GB RAM available for container
    - OpenAI API key or compatible provider

    <Callout type="info">
    Docker installation is recommended for evaluation and testing purposes.
    </Callout>

    ```bash
    # Pull the CUGA Docker image
    docker pull ibmresearch/cuga:latest

    # Run CUGA container
    docker run -it --rm \
      -p 8000:8000 \
      -e OPENAI_API_KEY=your_key_here \
      ibmresearch/cuga:latest
    ```
  </TabsContent>
  
  <TabsContent value="development">
    <h3>Development Setup</h3>
    <p>For contributors and developers working on CUGA.</p>

    **Requirements:**
    - Python 3.12+
    - UV package manager
    - Git
    - 5GB free storage (for dev dependencies)
    - OpenAI API key or compatible provider
    - Pre-commit (installed automatically)

    ```bash
    # Clone with submodules
    git clone --recursive https://github.com/cuga-project/cuga-agent.git
    cd cuga-agent

    # Install in development mode
    uv sync --dev

    # Install pre-commit hooks
    pre-commit install
    ```
  </TabsContent>
</Tabs>

## Troubleshooting

### Common Issues

<Callout type="warning">
Most installation issues are related to Python version, dependencies, or API key configuration.
</Callout>

| Issue | Solution |
|-------|----------|
| **Python version error** | Ensure Python 3.12+ is installed and in PATH |
| **Module not found** | Run `uv sync` to install dependencies |
| **API key errors** | Verify your `.env` file has correct API keys |
| **Port conflicts** | Check if ports 8000/8005 are available |
