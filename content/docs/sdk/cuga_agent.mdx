# CugaAgent Python SDK

CugaAgent can be easily integrated into your Python applications as a library. The SDK provides a clean, minimal API for creating and invoking agents with custom tools.

<Callout type="info">
All examples in this guide are tested and working. See the [integration tests](https://github.com/cuga-project/cuga-agent/blob/main/tests/integration/test_sdk_integration.py) for the full test suite.
</Callout>

## Installation

```bash
pip install cuga
# or
uv add cuga
```

## Quick Start

Create an agent with custom tools and invoke it with a task:

```python
from cuga import CugaAgent
from langchain_core.tools import tool

# Define your custom tools
@tool
def add_numbers(a: int, b: int) -> int:
    '''Add two numbers together'''
    return a + b

@tool
def multiply_numbers(a: int, b: int) -> int:
    '''Multiply two numbers together'''
    return a * b

# Create and invoke the agent
agent = CugaAgent(tools=[add_numbers, multiply_numbers])
result = await agent.invoke("What is 10 + 5?")
print(result)  # Agent uses add_numbers tool and returns "15"
```

## Core Concepts

### CugaAgent Class

The main interface for creating and using CugaAgent instances.

```python
from cuga import CugaAgent

agent = CugaAgent(
    tools=[...],          # List of LangChain tools
    tool_provider=None,   # Custom tool provider (advanced)
    model=None,           # Custom LLM (defaults to configured model)
    callbacks=None,       # LangChain callbacks
)
```

### Methods

#### `invoke()`

Invoke the agent with a message and get the response.

```python
result = await agent.invoke(
    message="What is 10 + 5?",      # str or List[BaseMessage]
    thread_id=None,                  # Optional thread ID for state isolation
    config=None,                     # Optional LangGraph config
)
```

#### `stream()`

Stream the agent's execution step by step for real-time monitoring.

```python
async for state in agent.stream("Calculate 20 + 22"):
    for node_name, node_state in state.items():
        if isinstance(node_state, dict) and 'script' in node_state:
            print(f"Executing: {node_state['script']}")
```

### State Isolation with Thread ID

Use `thread_id` for per-user state management and E2B sandbox caching:

```python
agent = CugaAgent(tools=[add_numbers])

# User 1's session
result1 = await agent.invoke(
    "What is 10 + 5?",
    thread_id="user-alice"
)

# User 2's session (isolated from User 1)
result2 = await agent.invoke(
    "What is 20 + 22?",
    thread_id="user-bob"
)
```

### Streaming Execution

Monitor agent execution in real-time:

```python
agent = CugaAgent(tools=[add_numbers, multiply_numbers])

async for state in agent.stream("Calculate 20 + 22"):
    for node_name, node_state in state.items():
        print(f"📍 Node: {node_name}")
        
        if isinstance(node_state, dict):
            # Observe code generation
            if 'script' in node_state and node_state['script']:
                print(f"💻 Code: {node_state['script']}")
            
            # Get final answer
            if 'final_answer' in node_state:
                print(f"✅ Answer: {node_state['final_answer']}")
```

### Dynamic Tool Management

Add tools after agent creation:

```python
agent = CugaAgent(tools=[add_numbers])

# First invocation
result1 = await agent.invoke("What is 8 + 9?")
print(result1)  # "17"

# Add new tool
agent.add_tool(multiply_numbers)

# Now can use the new tool
result2 = await agent.invoke("What is 4 * 5?")
print(result2)  # "20"
```

## Advanced Features

### Custom Tool Provider

For advanced scenarios, implement a custom tool provider:

```python
from cuga import CugaAgent
from cuga.backend.cuga_graph.nodes.cuga_lite.tool_provider_interface import (
    ToolProviderInterface,
    AppDefinition,
)

class MyToolProvider(ToolProviderInterface):
    def __init__(self, tools):
        self.tools = tools
        self.initialized = False
    
    async def initialize(self):
        self.initialized = True
    
    async def get_apps(self):
        return [
            AppDefinition(
                name="my_app",
                description="My custom application",
                type="api"
            )
        ]
    
    async def get_tools(self, app_name: str):
        if app_name == "my_app":
            return self.tools
        return []
    
    async def get_all_tools(self):
        return self.tools

# Use custom provider
tool_provider = MyToolProvider(tools=[add_numbers, multiply_numbers])
agent = CugaAgent(tool_provider=tool_provider)
result = await agent.invoke("Calculate 5 + 6")
```

### Custom Model Configuration

Use a custom LLM model:

```python
from cuga import CugaAgent
from cuga.backend.llm.models import LLMManager
from cuga.config import settings

# Get custom model
llm_manager = LLMManager()
custom_model = llm_manager.get_model(settings.agent.code.model)

# Create agent with custom model
agent = CugaAgent(tools=[add_numbers], model=custom_model)
result = await agent.invoke("What is 5 + 6?")
```

### LangGraph Integration

Access the underlying LangGraph for advanced use cases:

```python
from cuga import CugaAgent
from langgraph.checkpoint.memory import MemorySaver

agent = CugaAgent(tools=[add_numbers])

# Access the compiled LangGraph
compiled_graph = agent.graph

# Use with custom checkpointer
checkpointer = MemorySaver()

# You can now use the graph with LangGraph features
result = await compiled_graph.ainvoke(
    {"chat_messages": [HumanMessage(content="What is 2+2?")]},
    config={"configurable": {"thread_id": "user-123"}}
)
```

### Convenience Function

For one-off invocations without creating an agent instance:

```python
from cuga import run_agent

result = await run_agent(
    "What is 12 + 13?",
    tools=[add_numbers]
)
print(result)  # "25"
```

## Best Practices

<Steps>

<Step>
### Define Clear Tool Descriptions

The LLM reads tool docstrings to understand when to use them. Make them clear and specific:

```python
@tool
def get_weather(city: str) -> str:
    '''Get current weather information for a specific city.
    
    Args:
        city: Name of the city (e.g., "San Francisco", "New York")
    
    Returns:
        Weather description including temperature and conditions
    '''
    return f"Weather in {city}: Sunny, 72°F"
```

</Step>

<Step>
### Use Type Hints

Always include type hints for tool parameters:

```python
@tool
def calculate_discount(price: float, discount_percent: int) -> float:
    '''Calculate discounted price'''
    return price * (1 - discount_percent / 100)
```

</Step>

<Step>
### Handle Errors Gracefully

Return meaningful error messages from tools:

```python
@tool
def divide_numbers(a: float, b: float) -> str:
    '''Divide two numbers'''
    if b == 0:
        return "Error: Cannot divide by zero"
    return str(a / b)
```

</Step>

<Step>
### Use Thread IDs for State Isolation

When building multi-user applications, always use thread IDs:

```python
# Per-user isolation
result = await agent.invoke(
    message="What is my balance?",
    thread_id=f"user-{user_id}"
)
```

</Step>

</Steps>

## Configuration

CugaAgent uses [Dynaconf](https://www.dynaconf.com/) for configuration management, which allows you to configure the agent through both `settings.toml` files and environment variables.

### Configuration Hierarchy

Settings can be defined in multiple ways (in order of precedence):

1. **Environment Variables** (highest priority)
2. **`.env` file**
3. **`settings.toml` file**
4. **Default values** (lowest priority)

### Environment Variable Format

All settings in `settings.toml` can be overridden using environment variables with the format:

```
DYNACONF_<SECTION>__<SUBSECTION>__<KEY>=value
```

<Callout type="info">
Note the double underscore `__` between sections and single underscore `_` within names.
</Callout>

### Basic Environment Variables

Set these in your `.env` file:

```bash
# Required: LLM API key
OPENAI_API_KEY=sk-...

# Optional: Model configuration
AGENT_SETTING_CONFIG="settings.openai.toml"
MODEL_NAME=gpt-4o
```

### Advanced Configuration

#### E2B Sandbox Mode

Enable remote code execution in E2B sandboxes for enhanced security:

```bash
# Enable E2B sandbox for code execution
DYNACONF_ADVANCED_FEATURES__E2B_SANDBOX=true

# Set your E2B API key
E2B_API_KEY=your-e2b-api-key

# Optional: Configure E2B timeout (in seconds)
DYNACONF_ADVANCED_FEATURES__E2B_TIMEOUT=300
```

from cuga import CugaAgent
import os

#### Tool Shortlisting

When you have many tools, CugaAgent can automatically enable tool shortlisting to improve performance:

```bash
# Set threshold for automatic tool shortlisting (default: 10)
DYNACONF_ADVANCED_FEATURES__SHORTLISTING_TOOL_THRESHOLD=10
```

When the number of tools exceeds this threshold, CugaAgent will:
1. Use an LLM to identify relevant tools for the task
2. Only expose those tools to the agent
3. Reduce token usage and improve response time

<Callout type="tip">
Tool shortlisting is automatic! If you have more than 10 tools, CugaAgent will automatically use the `find_tools` mechanism to select the most relevant ones.
</Callout>

```python
from cuga import CugaAgent

# With 15 tools, shortlisting is automatically enabled
agent = CugaAgent(tools=[tool1, tool2, ..., tool15])

# CUGA will:
# 1. Analyze the user's task
# 2. Select the 4 most relevant tools
# 3. Execute with only those tools
result = await agent.invoke("Calculate something")
```

<Callout type="info">
**Note:** The CugaAgent SDK currently exposes the API-based agent for tool execution and code generation. Browser automation capabilities are not yet available through the SDK but will be added in a future release.
</Callout>

#### Step Limits

Control the maximum number of execution steps:

```bash
# Maximum steps before stopping (default: 20)
DYNACONF_ADVANCED_FEATURES__CUGA_LITE_MAX_STEPS=20
```

```python
from cuga import CugaAgent
import os

# Set custom step limit
os.environ["DYNACONF_ADVANCED_FEATURES__CUGA_LITE_MAX_STEPS"] = "30"

agent = CugaAgent(tools=[...])
# Agent will stop after 30 steps if task isn't complete
```

#### Security Settings

Configure code execution security:

```bash
# Execution timeout (seconds)
DYNACONF_ADVANCED_FEATURES__CODE_EXECUTION_TIMEOUT=60

# Enable Docker isolation (alternative to E2B)
DYNACONF_FEATURES__DOCKER_SANDBOX=true
```

### Programmatic Configuration

You can also configure settings programmatically:

```python
from cuga import CugaAgent
from cuga.config import settings
import os

# Override settings via environment
os.environ["DYNACONF_ADVANCED_FEATURES__E2B_SANDBOX"] = "true"
os.environ["DYNACONF_ADVANCED_FEATURES__SHORTLISTING_TOOL_THRESHOLD"] = "5"

# Access settings
print(f"E2B enabled: {settings.advanced_features.e2b_sandbox}")
print(f"Shortlisting threshold: {settings.advanced_features.shortlisting_tool_threshold}")

# Create agent (will use updated settings)
agent = CugaAgent(tools=[...])
```

### Configuration Reference

| Setting | Environment Variable | Default | Description |
|---------|---------------------|---------|-------------|
| E2B Sandbox | `DYNACONF_ADVANCED_FEATURES__E2B_SANDBOX` | `false` | Enable remote E2B sandbox execution |
| E2B API Key | `E2B_API_KEY` | - | Your E2B API key |
| Tool Threshold | `DYNACONF_ADVANCED_FEATURES__SHORTLISTING_TOOL_THRESHOLD` | `10` | Auto-enable shortlisting above this count |
| Max Steps | `DYNACONF_ADVANCED_FEATURES__CUGA_LITE_MAX_STEPS` | `20` | Maximum execution steps |
| Local Sandbox | `DYNACONF_FEATURES__LOCAL_SANDBOX` | `true` | Use local sandbox vs E2B |
| Code Timeout | `DYNACONF_ADVANCED_FEATURES__CODE_EXECUTION_TIMEOUT` | `60` | Code execution timeout (seconds) |

## API Reference

### CugaAgent

```python
class CugaAgent:
    def __init__(
        self,
        tools: Optional[List[BaseTool]] = None,
        tool_provider: Optional[ToolProviderInterface] = None,
        model: Optional[BaseChatModel] = None,
        callbacks: Optional[List[BaseCallbackHandler]] = None,
    )
    
    async def invoke(
        self,
        message: Union[str, List[BaseMessage]],
        thread_id: Optional[str] = None,
        config: Optional[Dict[str, Any]] = None,
    ) -> str
    
    async def stream(
        self,
        message: Union[str, List[BaseMessage]],
        thread_id: Optional[str] = None,
        config: Optional[Dict[str, Any]] = None,
    )
    
    def add_tool(self, tool: BaseTool)
    
    def add_tools(self, tools: List[BaseTool])
    
    @property
    def graph(self) -> CompiledGraph
```

### run_agent()

```python
async def run_agent(
    message: str,
    tools: Optional[List[BaseTool]] = None,
    model: Optional[BaseChatModel] = None,
) -> str
```

## Troubleshooting

### Import Errors

If you get import errors, ensure CugaAgent is installed:

```bash
pip install cuga
# or
uv sync  # if developing from source
```

### Model Configuration

If the agent fails to initialize, check your `.env` file:

```bash
# Verify environment variables are set
cat .env | grep OPENAI_API_KEY
```

### Tool Execution Errors

If tools aren't being called:

- ✅ Check tool docstrings are clear and descriptive
- ✅ Ensure tool parameters have type hints
- ✅ Verify tools return string or JSON-serializable values
- ✅ Check logs for security validation errors

<Callout type="warning">
CugaAgent requires that all generated code contains at least one `print()` statement. This is enforced by the security validator.
</Callout>
