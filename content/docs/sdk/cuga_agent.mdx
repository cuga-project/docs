# CugaAgent

The `CugaAgent` class provides a simple, high-level interface for creating and running CUGA agents. It wraps the underlying LangGraph execution complexity, offering an easy way to manage tools, handle conversations, and integrate policies.

## Initialization

<Steps>
<Step>
### Import
Import the agent and necessary tools from `cuga.sdk` and `langchain_core`.

```python
from cuga.sdk import CugaAgent
from langchain_core.tools import tool
```
</Step>

<Step>
### Define Tools
Create the tools your agent will use.

```python
@tool
def get_weather(city: str) -> str:
    """Get weather for a city"""
    return f"Weather in {city}: Sunny, 72°F"
```
</Step>

<Step>
### Create Agent
Initialize the `CugaAgent` with your tools and optional model configuration.

```python
agent = CugaAgent(
    tools=[get_weather],
    # model=... (optional)
)
```
</Step>
</Steps>

## Core Methods

### `invoke`

Run the agent with a message.

<Tabs items={['Single Turn', 'Multi-turn / Resume']}>
<Tab value="Single Turn">
```python
result = await agent.invoke("What is the weather in Tokyo?")
print(result)
```
</Tab>
<Tab value="Multi-turn / Resume">
```python
# Multi-turn with list of messages
result = await agent.invoke([
    HumanMessage(content="My name is Alice"),
    AIMessage(content="Hello Alice"),
    HumanMessage(content="What is my name?")
])

# Resume execution (e.g. after tool approval)
result = await agent.invoke(
    None,
    thread_id="user-123",
    action_response=approval_response
)
```
</Tab>
</Tabs>

**Signature Parameters**

### `stream`

Stream the agent's execution step-by-step.

```python
async for state in agent.stream("Analyze this dataset"):
    print(state)
```

**Signature Parameters**

### `add_tool` / `add_tools`

Dynamically add tools to the agent instance.

```python
agent.add_tool(new_tool)
# or
agent.add_tools([tool1, tool2])
```

<Accordions>
<Accordion title="Important Note on Graph Recreation">
Replacing or adding tools will trigger a recreation of the underlying graph on the next invocation. This is only supported when using the default `DirectLangChainToolsProvider`.
</Accordion>
</Accordions>

## Properties

### `graph`

Access the underlying compiled LangGraph `StateGraph`.

```python
compiled_graph = agent.graph
# Use the graph directly
await compiled_graph.ainvoke(...)
```

### `policies`

Access the `PoliciesManager`. See [Policies](../policies).

## Helper Functions

### `run_agent`

A convenience function to quickly spin up a new agent and run a single turn.

```python
from cuga.sdk import run_agent

result = await run_agent(
    "Calculate 5 * 10",
    tools=[calculator_tool]
)
```
