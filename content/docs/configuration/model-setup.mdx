IBM CUGA supports multiple LLM providers, allowing you to choose the best option for your needs and budget.

## 🌐 Supported Providers

| Provider | Best For | Cost | Setup Complexity |

| **Azure OpenAI** | Azure Users | Pay-per-use | ⭐⭐ Medium |

## 🏢 ETE LiteLLM (Recommended for IBM Teams)

The default and recommended configuration for IBM team members.

### Setup Instructions

1. **Access Portal**
   - Visit [ETE LiteLLM Portal](https://ete-litellm.bx.cloud9.ibm.com/ui/)
   - Login with your IBM credentials

2. **Generate API Key**
   - Navigate to the API keys section
   - Generate a new API key
   - Copy the key (starts with `ete-`)

3. **Configure Environment**
   ```bash
   # Edit .env file
   OPENAI_API_KEY=ete-your-key-here
   AGENT_SETTING_CONFIG="settings.litellm.toml"
   ```

4. **Verify Configuration**
   ```bash
   uv run test_sandbox
   ```

### Benefits

- ✅ **Free for IBM employees**
- ✅ **High performance and reliability**
- ✅ **IBM security compliance**
- ✅ **Seamless integration**

## 🌐 OpenAI

Popular choice for general use with excellent model quality.

### Setup Instructions

1. **Create Account**
   - Visit [platform.openai.com](https://platform.openai.com)
   - Sign up or log in to your account

2. **Generate API Key**
   - Navigate to [API Keys](https://platform.openai.com/api-keys)
   - Click "Create new secret key"
   - Copy the key (starts with `sk-`)

3. **Configure Environment**
   ```bash
   # Edit .env file
   OPENAI_API_KEY=sk-your-key-here
   AGENT_SETTING_CONFIG="settings.openai.toml"
   ```

4. **Verify Configuration**
   ```bash
   uv run test_sandbox
   ```

### Cost Considerations

- **GPT-4**: ~$0.03 per 1K input tokens, ~$0.06 per 1K output tokens
- **GPT-3.5-turbo**: ~$0.0015 per 1K input tokens, ~$0.002 per 1K output tokens

## 🔵 IBM WatsonX

Enterprise-grade AI platform with IBM's latest models.

### Setup Instructions

1. **Access WatsonX**
   - Visit [IBM WatsonX](https://www.ibm.com/watsonx)
   - Create a new project or access existing one

2. **Get Credentials**
   - **Project ID**: Found in project settings
   - **API Key**: Generate in project credentials
   - **Region/URL**: Your deployment region

3. **Configure Environment**
   ```bash
   # Edit .env file
   WATSONX_API_KEY=your-watsonx-api-key
   WATSONX_PROJECT_ID=your-project-id
   WATSONX_URL=https://us-south.ml.cloud.ibm.com
   AGENT_SETTING_CONFIG="settings.watsonx.toml"
   ```

4. **Verify Configuration**
   ```bash
   uv run test_sandbox
   ```

### Benefits

- ✅ **Enterprise security**
- ✅ **IBM's latest AI models**
- ✅ **Compliance features**
- ✅ **Scalable infrastructure**

## ☁️ Azure OpenAI

Microsoft's managed OpenAI service for Azure users.

### Setup Instructions

1. **Azure Setup**
   - Access [Azure Portal](https://portal.azure.com)
   - Create or access Azure OpenAI resource
   - Get endpoint URL and API key

2. **Configure Environment**
   ```bash
   # Edit .env file
   AGENT_SETTING_CONFIG="settings.azure.toml"
   AZURE_OPENAI_API_KEY="<your azure apikey>"
   AZURE_OPENAI_ENDPOINT="<your azure endpoint>"
   OPENAI_API_VERSION="2024-08-01-preview"
   ```

3. **Verify Configuration**
   ```bash
   uv run test_sandbox
   ```

### Benefits

- ✅ **Azure integration**
- ✅ **Enterprise security**
- ✅ **Managed service**
- ✅ **Compliance certifications**

## ⚙️ Configuration Files

### Settings Structure

```
configurations/
├── settings.litellm.toml      # Default ETE configuration
├── settings.openai.toml       # OpenAI configuration
├── settings.watsonx.toml      # WatsonX configuration
├── settings.azure.toml        # Azure configuration
└── modes/
    ├── fast.toml              # Fast mode settings
    ├── accurate.toml          # Accurate mode settings
    └── custom.toml            # Custom mode settings
```

### Example Configuration Files

#### settings.litellm.toml (Default)
```toml
[llm]
provider = "litellm"
model = "gpt-4"
temperature = 0.1
max_tokens = 4000

[litellm]
base_url = "https://ete-litellm.bx.cloud9.ibm.com"
api_key = "${OPENAI_API_KEY}"
```

#### settings.openai.toml
```toml
[llm]
provider = "openai"
model = "gpt-4"
temperature = 0.1
max_tokens = 4000

[openai]
api_key = "${OPENAI_API_KEY}"
```

## 🔄 Switching Between Providers

### Method 1: Environment Variable

```bash
# Switch to OpenAI
export AGENT_SETTING_CONFIG="settings.openai.toml"

# Switch to WatsonX
export AGENT_SETTING_CONFIG="settings.watsonx.toml"

# Switch back to ETE (default)
export AGENT_SETTING_CONFIG="settings.litellm.toml"
```

### Method 2: Edit .env File

```bash
# Edit .env file
AGENT_SETTING_CONFIG="settings.openai.toml"
```

### Method 3: Command Line

```bash
# Start with specific config
cuga start demo --config settings.openai.toml
```

## 🔍 Troubleshooting

### Common Issues

| Issue                    | Solution                                    |
| ------------------------ | ------------------------------------------- |
| 🔴 API key invalid       | Verify key format and permissions           |
| 🔴 Rate limit exceeded   | Check usage limits and billing              |
| 🔴 Model not found       | Verify model name in configuration          |
| 🔴 Network timeout       | Check firewall and proxy settings           |
| 🔴 Authentication failed | Verify API key and endpoint URL             |

### Debug Commands

```bash
# Check current configuration
echo $AGENT_SETTING_CONFIG

# Verify environment variables
cat .env | grep -E "(OPENAI_API_KEY|WATSONX|AZURE)"

# Test API connectivity
curl -H "Authorization: Bearer $OPENAI_API_KEY" \
     https://api.openai.com/v1/models
```
