Fine-tune CUGA's behavior through comprehensive configuration options, including execution modes, agent behavior, and advanced features.

## 🎯 Configuration Overview

CUGA uses TOML configuration files for flexible and readable settings management:

```
configurations/
├── settings.litellm.toml      # Default ETE configuration
├── settings.openai.toml       # OpenAI configuration
├── settings.watsonx.toml      # WatsonX configuration
├── settings.azure.toml        # Azure configuration
└── modes/
    ├── fast.toml              # Fast mode settings
    ├── accurate.toml          # Accurate mode settings
    └── custom.toml            # Custom mode settings
```

## 🚀 Execution Modes

### Mode Selection

CUGA offers three execution modes optimized for different use cases:

| Mode       | Speed | Accuracy | Use Case                    |
|------------|-------|----------|-----------------------------|
| **Fast**   | ⭐⭐⭐⭐⭐ | ⭐⭐⭐    | Development, testing        |
| **Accurate** | ⭐⭐⭐ | ⭐⭐⭐⭐⭐ | Production, critical tasks  |
| **Custom** | ⭐⭐⭐⭐ | ⭐⭐⭐⭐ | Tailored workflows          |

### Fast Mode Configuration

```toml
# configurations/modes/fast.toml
[mode]
name = "fast"
description = "Optimized for speed and development"

[agent]
temperature = 0.1
max_tokens = 2000
timeout = 60

[execution]
max_retries = 2
parallel_tasks = 4
cache_enabled = true
```

### Accurate Mode Configuration

## 🔧 Core Settings

### Agent Configuration

```toml
[agent]
# LLM Settings
temperature = 0.1              # 0.0 to 1.0 (lower = more deterministic)
max_tokens = 4000              # Maximum tokens per response
top_p = 0.9                    # Nucleus sampling parameter
frequency_penalty = 0.0        # Reduce repetition
presence_penalty = 0.0         # Encourage new topics

# Execution Settings
timeout = 120                  # Seconds per task
max_retries = 3                # Retry attempts on failure
retry_delay = 5                # Seconds between retries

# Memory Settings
memory_enabled = true          # Enable conversation memory
memory_size = 1000             # Number of interactions to remember
memory_ttl = 3600              # Memory time-to-live in seconds
```

### Feature Configuration

```toml
[features]
# Execution Mode
mode = "hybrid"                # api, chat, or hybrid
cuga_mode = "accurate"         # fast, accurate, or custom

# API Integration
api_enabled = true             # Enable API mode
chat_enabled = true            # Enable chat mode
browser_enabled = true         # Enable browser automation

# Advanced Features
save_reuse_enabled = true      # Enable workflow persistence
experiment_tracking = true     # Enable experiment logging
parallel_execution = true      # Enable parallel task execution
```

### Advanced Features

```toml
[advanced_features]
# Benchmark Configuration
benchmark = "appworld"         # webarena, appworld, or custom

# Performance Settings
cache_size = 1000              # Number of cached responses
max_workers = 4                # Maximum parallel workers
connection_pool_size = 10      # HTTP connection pool size

# Security Settings
ssl_verify = true              # Verify SSL certificates
api_key_rotation = false       # Enable automatic API key rotation
audit_logging = true           # Log all API interactions
```

## 🎮 Mode Switching

### Method 1: Environment Variable

```bash
# Switch to fast mode
export CUGA_MODE="fast"

# Switch to accurate mode
export CUGA_MODE="accurate"

# Switch to custom mode
export CUGA_MODE="custom"
```

### Method 2: Settings File

Edit `settings.toml`:

```toml
[features]
cuga_mode = "fast"  # or "accurate" or "custom"
```

### Method 3: Command Line

```bash
# Start with specific mode
cuga start demo --mode fast
cuga start demo --mode accurate
cuga start demo --mode custom
```

## 🔄 Dynamic Configuration

### Runtime Configuration Changes

CUGA supports dynamic configuration updates:

```python
# Update configuration at runtime
from cuga.config import Config

config = Config()
config.update({
    'agent.temperature': 0.2,
    'execution.max_retries': 5
})
```

### Configuration Validation

```bash
# Validate configuration
cuga config validate

# Check current settings
cuga config show

# Reset to defaults
cuga config reset
```

## 📊 Performance Tuning

### Memory Optimization

```toml
[performance]
# Memory Management
max_memory_usage = "8GB"       # Maximum memory usage
garbage_collection = "auto"    # GC strategy
memory_pool_size = 1000        # Object pool size

# Caching
cache_strategy = "lru"         # LRU, LFU, or FIFO
cache_ttl = 3600              # Cache time-to-live
cache_max_size = 10000        # Maximum cache entries
```

### Execution Optimization

```toml
[execution]
# Parallel Processing
max_concurrent_tasks = 4       # Maximum parallel tasks
task_queue_size = 100          # Task queue capacity
worker_timeout = 300           # Worker timeout in seconds

# Resource Management
cpu_limit = 4                  # CPU core limit
memory_limit = "8GB"           # Memory limit per task
disk_quota = "1GB"             # Disk usage limit
```

## 🧪 Testing Configuration

```bash
cuga benchmark --compare fast,accurate,custom
```

## 🔍 Configuration Debugging

### Debug Mode

Enable debug logging:

```toml
[logging]
level = "DEBUG"
format = "detailed"
output = "file"
file_path = "logs/cuga_debug.log"

[debug]
enabled = true
verbose = true
trace_execution = true
log_prompts = true
```

### Configuration Inspection

```bash
# Show current configuration
cuga config show

# Show specific section
cuga config show --section agent
cuga config show --section features

# Export configuration
cuga config export --format json
cuga config export --format yaml
```

## 🚨 Error Handling

### Retry Configuration

```toml
[retry]
# General Retry Settings
max_retries = 3                # Maximum retry attempts
retry_delay = 5                # Base delay in seconds
backoff_multiplier = 2.0       # Exponential backoff multiplier
max_delay = 60                 # Maximum delay between retries

# Specific Error Handling
retry_on_timeout = true        # Retry on timeout errors
retry_on_rate_limit = true     # Retry on rate limit errors
retry_on_network_error = true  # Retry on network errors
```

### Fallback Configuration

```toml
[fallback]
# Model Fallback
primary_model = "gpt-4"
fallback_model = "gpt-3.5-turbo"
fallback_on_error = true

# Provider Fallback
primary_provider = "openai"
fallback_provider = "litellm"
fallback_on_failure = true
```

## 🔄 Configuration Migration

### Version Compatibility

```bash
# Check configuration version
cuga config version

# Migrate configuration
cuga config migrate --from v1.0 --to v2.0

# Backup configuration
cuga config backup --output config_backup.toml
```

### Import/Export

```bash
# Export configuration
cuga config export --output my_config.toml

# Import configuration
cuga config import --file my_config.toml

# Merge configurations
cuga config merge --base default.toml --override custom.toml
```
