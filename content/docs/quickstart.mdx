# Quick Start Guide

Get IBM CUGA up and running on your system in just a few minutes. This guide will walk you through the essential setup steps.

## Prerequisites

Before you begin, ensure you have the following installed:

- **Python 3.12** or higher
- **UV** package manager
- **Git** for cloning the repository
- **OpenAI API Key** (or compatible LLM provider)

<Callout type="info">
If you don't have UV installed, follow the [UV installation guide](https://docs.astral.sh/uv/getting-started/installation/).
</Callout>

## Installation Steps

<Steps>
  <Step title="Clone Repository">
    Clone the CUGA repository to your local machine
  </Step>
  <Step title="Setup Environment">
    Create a virtual environment and install dependencies
  </Step>
  <Step title="Configure API Keys">
    Set up your environment variables and API keys
  </Step>
  <Step title="Test Installation">
    Verify everything is working correctly
  </Step>
</Steps>

## Step 1: Clone the Repository

```bash
git clone git@github.ibm.com:research-rpa/cuga.git
cd cuga
```

## Step 2: Setup Environment

Create a virtual environment and install dependencies:

```bash
uv venv --python=3.12
source .venv/bin/activate  # On Windows: .venv\Scripts\activate
uv sync
```

<Callout type="warning">
Make sure to activate the virtual environment before running any CUGA commands.
</Callout>

## Step 3: Configure Environment

Copy the example environment file and add your API key:

```bash
cp .env.example .env
```

Edit the `.env` file and add your OpenAI API key:

```bash
OPENAI_API_KEY=your_api_key_here
```

<Callout type="info">
For IBM team members, you can use the ETE LiteLLM API key from the [ETE LiteLLM portal](https://ete-litellm.bx.cloud9.ibm.com/ui/).
</Callout>

## Step 4: Test the Installation

Verify that CUGA is working correctly:

```bash
uv run test_sandbox
```

If everything is set up correctly, you should see a success message.

## First Run

Start CUGA in demo mode to explore its capabilities:

```bash
cuga start demo
```

Try this example query to test the system:

<CodeBlock lang="bash">
"get my top account by revenue from digital sales"
</CodeBlock>

## Troubleshooting

If you encounter issues during installation:

1. **Check Python version**: Ensure you're using Python 3.12+
2. **Verify virtual environment**: Make sure it's activated
3. **Check API key**: Verify your OpenAI API key is valid
4. **Review logs**: Check for error messages in the terminal
