Try These API Queries

These queries work perfectly in API-only mode:

  • "get my top account by revenue from digital sales"
  • "analyze revenue trends for the last quarter"
  • "calculate total sales by region"
  • "find accounts with revenue above $100,000"
  • "get contact information for account ABC123"

Note: In API-only mode, CUGA cannot interact with this web page. It will only make API calls and return the results in the chat interface.

What CUGA Does in API Mode

When you ask CUGA to get data, it will:

  1. Parse your request: Understand what data you need
  2. Make API calls: Query the Digital Sales API
  3. Process the response: Analyze and format the data
  4. Return results: Provide the information in the chat

What it cannot do:

  • Navigate to web pages
  • Click buttons or fill forms
  • Take screenshots
  • Interact with browser elements

API Only Mode Benefits

  • 🚀 Faster Execution: No browser overhead
  • 💾 Lower Resource Usage: Minimal memory and CPU
  • 🖥️ Server Deployment: Can run on headless servers
  • 📈 Better Scalability: Ideal for high-volume operations
  • 🔒 Higher Reliability: No browser-related failures

Digital Sales API Endpoints

The demo environment includes these API endpoints:

GET /api/accounts - List all accounts GET /api/accounts/{id} - Get specific account GET /api/accounts/top-revenue - Get top revenue accounts GET /api/accounts/by-industry/{industry} - Get accounts by industry GET /api/accounts/by-location/{location} - Get accounts by location GET /api/analytics/revenue-trends - Get revenue trends GET /api/analytics/sales-by-region - Get sales by region GET /api/contacts/{account_id} - Get contacts for account

Testing API Only Mode

To test this mode:

  1. Start CUGA with --mode api or set CUGA_MODE="api"
  2. Ask CUGA one of the example queries above
  3. CUGA will make API calls and return the results
  4. Note that it cannot modify this web page

Expected behavior: CUGA will respond with API data in the chat interface, but this web page will remain unchanged.

Configuration

To enable API-only mode, set in your settings.toml:

[features] mode = "api" [api_mode] enabled = true browser_enabled = false web_automation = false

Or use environment variable:

export CUGA_MODE="api"
