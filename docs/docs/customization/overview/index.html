🚀 Reasoning Modes

Mode Selection

CUGA offers three reasoning modes optimized for different use cases:

ModeSpeedAccuracyUse Case
Fast⭐⭐⭐⭐⭐⭐⭐⭐Development, testing
Balanced⭐⭐⭐⭐⭐⭐⭐⭐General purpose, workflows
Accurate⭐⭐⭐⭐⭐⭐⭐⭐Production, critical tasks

Fast Mode Configuration

Fast Mode Features:

  • Speed Optimized: Minimal task decomposition for quick execution
  • Direct Execution: Bypasses complex reasoning chains
  • Local Sandbox: Uses local execution environment
  • No Reflection: Skips output validation for speed
  • Quick Code Generation: Fast code generation without extensive validation

Balanced Mode Configuration

Balanced Mode Features:

  • Task Decomposition: Breaks complex tasks into manageable steps
  • Reflection Enabled: Validates code output for quality
  • Local Sandbox: Uses local execution environment
  • No Chat Interface: Focuses on API-based execution
  • Fast Code Generation: Quick code generation with validation

Accurate Mode Configuration

Accurate Mode Features:

  • Full Task Decomposition: Comprehensive task breakdown
  • Thought Process: Enables detailed reasoning chains
  • Code Output Summary: Provides detailed execution summaries
  • Accurate Code Generation: High-quality code with extensive validation
  • Save & Reuse: Enables workflow persistence
  • Multiple Reflections: Multiple validation passes for accuracy
